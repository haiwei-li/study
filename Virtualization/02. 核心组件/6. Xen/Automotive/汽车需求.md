# 计算

> Computing

- **C1**: **静态资源分区**以及灵活的**按需资源分配**(包括 `CPU, RAM, GPU 和 IO`).

- **C2**: **内存总线带宽** / **IO 总线带宽**的分配与重新平衡.

# 外设共享

> Peripherals sharing

- **P1**: **GPU** 和**显示器**应能在不同执行环境之间共享, 需同时支持**固定配置** (每个执行环境连接到**自己的显示器**, 或连接到**单个显示器**上的**指定区域**) 和**灵活配置**(不同执行环境的界面形状, 层级顺序, 位置和分配可在运行时改变).

- **P2**: 输入设备应能根据当前模式, 显示配置 (针对触摸屏), 活跃应用程序(针对旋钮和按钮) 等情况, 将输入路由到一个或多个执行环境.

- **P3**: 音频应能在不同执行环境之间共享. 需支持针对多个音频流的复杂混音策略, 以及动态源 / 宿设备 (如蓝牙配置文件, USB 扬声器 / 麦克风等) 的路由.

- **P4**: 网络应能在不同执行环境之间共享. 需支持具有不同安全特性的虚拟网络(例如, 流量过滤和安全机制).

- **P5**: 存储应支持静态或共享分配, 同时支持动态存储设备 (如 USB 大容量存储设备) 的路由.

# 安全

> Security

- **SE1**: 所有执行环境都应支持**信任根**(Root of Trust)和安全启动.

- **SE2**: 所有执行环境都应具备并可配置可信计算(如独立的可信平台模块, ARM TrustZone 或类似技术).

- **SE3**: 应支持硬件隔离(如缓存, 中断, 输入输出内存管理单元, 防火墙等).

- **SE4**: 应支持安全更新.

# 性能和功耗

> Performance and Power consumption

- **PP1**: 虚拟化的性能开销应降至最低: 在 CPU / 内存基准测试中为 1 - 2%, 在 GPU 基准测试中最高为 5%.

- **PP2**: 应保证性能的可预测性. 在任何情况下 (如意外事件, 系统过载等) 都应满足最低性能要求.

- **PP3**: 执行环境应能**快速启动**: **安全关键型应用程序**的启动时间应少于 2 秒, **仪表盘的启动时间**应少于 5 秒, **车载信息娱乐系统**的启动时间应少于 10 秒. 应支持休眠和内存挂起功能.

- **PP4**: 执行环境的启动顺序应具有可预测性.

- **PP5**: 应为每个执行环境实施具有灵活策略的高级电源管理.

# 安全保障

> Safety

- **SA1**: 应支持系统监控, 以证明和验证系统是否正常运行.

- **SA2**: 每个执行环境在出现故障时应能够重启.

- **SA3**: 应支持冗余机制, 以实现最高级别的容错能力, 并在出现故障时有后备解决方案可供响应.

- **SA4**: 应保证实时支持以及可预测的响应时间.
