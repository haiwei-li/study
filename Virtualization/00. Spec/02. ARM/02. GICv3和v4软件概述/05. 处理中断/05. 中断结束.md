
当中断处理完成后, 软件必须通知中断控制器该中断已被处理, 以便状态机可以转换到下一个状态. GICv3 架构将此视为两个任务:
-**优先级降低**: 即将运行优先级降低回处理中断之前的值.
-**停用**: 即更新当前正在处理的中断的状态机. 通常, 这将是从活动状态转换到非活动状态.

在 GICv3 架构中, 优先级降低和停用可以同时发生, 也可以分别发生, 这取决于 ICC_CTLR_ELn.EOImode 的设置.
 - EOImode = 0: 向 ICC_EOIR0_EL1 写入组 0 中断, 或向 ICC_EOIR1_EL1 写入组 1 中断, 将同时执行优先级降低和停用操作. 这是简单裸机环境中通常使用的模式.
 - EOImode = 1: 向 ICC_EOIR0_EL1 写入组 0 中断, 或向 ICC_EOIR1_EL1 写入组 1 中断, 只会导致优先级降低. 需要单独向 ICC_DIR_EL1 写入来执行停用操作. 这种模式通常用于虚拟化目的.

在写入这些寄存器时, 软件必须写入 INTID.
